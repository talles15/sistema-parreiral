# 🚀 Como Configurar no GitHub Pages

Siga este guia passo-a-passo para hospedar seu Sistema de Custos do Parreiral gratuitamente no GitHub Pages.

## 📋 Pré-requisitos

- Conta no GitHub (gratuita)
- Conhecimento básico de Git (opcional, pode usar interface web)

## 🎯 Passo a Passo

### 1. 📂 Criar Repositório

1. Acesse [GitHub](https://github.com)
2. Clique em **"New repository"** (botão verde)
3. Configure o repositório:
   - **Repository name**: `sistema-parreiral` (ou nome de sua escolha)
   - **Description**: `Sistema completo para calcular custos de aplicações em parreiral`
   - ✅ **Public** (necessário para GitHub Pages gratuito)
   - ✅ **Add README file**
   - ✅ **Add .gitignore** → selecione "None"
   - **License**: MIT License (recomendado)
4. Clique em **"Create repository"**

### 2. 📁 Estrutura de Arquivos

Crie os seguintes arquivos no seu repositório:

```
sistema-parreiral/
├── index.html          # Arquivo principal (código HTML completo)
├── sw.js               # Service Worker para PWA
├── manifest.json       # Web App Manifest
├── README.md           # Documentação
├── LICENSE             # Licença MIT
└── docs/               # Pasta para documentação
    └── images/         # Screenshots (opcional)
```

### 3. 📝 Adicionar Arquivos

#### Método A: Interface Web do GitHub (Mais Fácil)

1. No seu repositório, clique em **"Add file" → "Create new file"**
2. Digite `index.html` como nome
3. Cole todo o código HTML do sistema
4. Role para baixo e clique em **"Commit new file"**
5. Repita para os outros arquivos:
   - `sw.js` (Service Worker)
   - `manifest.json` (Web App Manifest)

#### Método B: Git Command Line

```bash
# Clonar o repositório
git clone https://github.com/SEU-USUARIO/sistema-parreiral.git
cd sistema-parreiral

# Adicionar arquivos
# (copie os arquivos para a pasta)

# Commit e push
git add .
git commit -m "🍇 Adicionar Sistema de Custos do Parreiral v2.0"
git push origin main
```

### 4. ⚙️ Configurar GitHub Pages

1. No seu repositório, vá em **Settings** (última aba)
2. Role até a seção **"Pages"** (menu lateral esquerdo)
3. Em **"Source"**, selecione:
   - **Deploy from a branch**
   - **Branch**: `main` (ou `master`)
   - **Folder**: `/ (root)`
4. Clique em **"Save"**
5. ✅ GitHub mostrará: "Your site is published at https://seu-usuario.github.io/sistema-parreiral/"

### 5. 🌐 Acessar o Sistema

Após alguns minutos, seu sistema estará disponível em:
```
https://SEU-USUARIO.github.io/sistema-parreiral/
```

Substitua `SEU-USUARIO` pelo seu username do GitHub.

## 🔧 Configurações Adicionais

### Custom Domain (Opcional)
Se você tem um domínio próprio:

1. Na seção **Pages** do Settings
2. Em **"Custom domain"**, digite: `meudominio.com`
3. ✅ **Enforce HTTPS** (recomendado)
4. Configure o DNS do seu domínio:
   - Adicione um CNAME apontando para `seu-usuario.github.io`

### Atualizações Automáticas
Para atualizar o sistema:

1. Edite os arquivos no GitHub (interface web)
2. Ou faça push de novas versões via Git
3. O site será atualizado automaticamente em alguns minutos

## 🐛 Troubleshooting

### Site não carrega
- ✅ Verifique se o repositório é **público**
- ✅ Confirme que o arquivo se chama exatamente `index.html`
- ✅ Aguarde até 10 minutos após configurar

### Erro 404
- ✅ Verifique se GitHub Pages está ativado
- ✅ Confirme a branch correta (main/master)
- ✅ Verifique se o arquivo `index.html` está na raiz do repositório

### PWA não instala
- ✅ Confirme que `manifest.json` e `sw.js` estão no repositório
- ✅ Verifique se o site usa HTTPS (GitHub Pages tem por padrão)
- ✅ Teste em Chrome/Edge para melhor suporte a PWA

### Service Worker não funciona
- ✅ Abra DevTools → Application → Service Workers
- ✅ Verifique erros no console
- ✅ Force refresh com Ctrl+F5
